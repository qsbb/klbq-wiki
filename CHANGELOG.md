# 更新日志

## 1.6.0

- **新增图片缓存功能，解决角色图片加载不出来的问题**
  - 查询过的角色立绘、皮肤图会缓存到本地 `data/images/` 目录
  - 下次查询同一图片直接读取本地文件，不再重复网络下载
  - 适用于所有图片场景：角色卡片立绘、皮肤详情图、生日卡片立绘
  - 图片 URL 经 MD5 哈希作为本地文件名，自动推断扩展名
  - 支持批量并发下载（`getMany`，默认并发 4）
- **新增配置项**
  - `image_cache`（默认 `true`）：图片缓存开关
  - `image_cache_ttl`（默认 `30`，范围 0-365 天）：缓存有效期，0 表示永不过期
  - 可通过 `-设置 image_cache off` 关闭缓存
  - 可通过 `-设置 image_cache_ttl 0` 设为永不过期
- **容错机制**
  - 缓存目录创建失败时自动降级为直接使用远程 URL
  - 单张图片下载失败时回退到远程 URL，不影响其他图片
  - `logger` 全局变量不存在时静默处理（测试环境兼容）
- **`.gitignore` 新增 `data/` 规则**：缓存数据不纳入版本控制
- **新增 `lib/image-cache.js`**：独立的图片缓存模块，可复用

## 1.4.1

- **设置卡片配置项改为中文显示**
  - 每个 CONFIG_META 配置项新增 `label` 字段（中文显示名）
  - 设置卡片显示格式：`中文label（英文key）`，如 `图片渲染（render_image）`
  - 英文 key 仍用于 `-设置 <key> <值>` 修改命令
- **修复设置卡片"使用方法"分组显示奇怪字符的问题**
  - 根因：代码中手动调用 `escapeHtml` 转义后，art-template 的 `{{}}` 又自动转义一次，导致双重转义
  - `<项名>` 被转义成 `&lt;项名&gt;` 后又被转义成 `&amp;lt;项名&amp;gt;`，图片上显示为 `&lt;项名&gt;` 字面字符
  - 修复：移除 `renderImage` 和 `renderHelp` 中所有手动 `escapeHtml` 调用，统一由 art-template 自动转义
  - 修复后 `<项名>` `<值>` 等尖括号内容正确显示

## 1.3.2

- 修复 `#klbq心夏`、`#卡拉彼丘心夏`、`#卡丘心夏` 等无空格前缀写法无法触发查询的问题
  - 原因：主正则 `(?:\s|$)` 要求前缀后必须有空白或字符串结尾，导致 `#klbq` 紧跟中文时不匹配
  - 修复：移除 `(?:\s|$)` 限制，前缀后紧跟关键词也可识别
  - 同步修复 `extractPrefix` 函数的 `(?=\s|$)` 前瞻，确保无空格写法的提示前缀也正确
- 现在前缀与关键词之间有无空格均可识别：
  - `#klbq 心夏` 和 `#klbq心夏` 均可
  - `#卡拉彼丘 心夏` 和 `#卡拉彼丘心夏` 均可
  - `#卡丘 心夏` 和 `#卡丘心夏` 均可
  - `/` 前缀同理

## 1.3.1

- 修复 `-帮助` 等单条内容图片卡片右侧大片空白的问题

  - 影响：帮助、赛季、生日等单条内容卡片现在占满整行
- 修复角色/武器卡片底部提示硬编码 `#klbq` 的问题

  - 例如 `-心夏` → `提示：可继续使用 - 空境 查询心夏的武器`
  - 例如 `#卡拉彼丘 心夏` → `提示：可继续使用 #卡拉彼丘 空境 查询心夏的武器`

## 1.3.0

- 新增 `-卡拉彼丘更新` / `-卡拉彼丘强制更新` 自动重启功能
  - 更新成功后自动重启 Yunzai，无需手动操作
  - 采用 Yunzai 官方重启机制：redis 设置 `Yz:restart` 标记 + `process.exit`
  - 依赖 PM2 等进程管理器自动拉起新进程
  - 默认延时 3 秒后重启，确保消息发送完成
- 新增配置项 `auto_restart`（默认 `true`）：可关闭自动重启改为手动
- 新增配置项 `restart_delay`（默认 `3`，范围 1-30）：重启前等待秒数
- 可通过 `-设置 auto_restart off` 关闭自动重启
- 可通过 `-设置 restart_delay 5` 修改等待时间
